## 项⽬概述
本项目基于MuJoCo物理引擎与MPC模型预测控制算法，开发了一套汽车仪表盘仿真系统。系统可实时模拟车辆行驶过程中速度、转速、油量、水温等核心参数的动态显示，支持自定义车辆工况与故障模拟场景。通过Ubuntu环境下的C/C++开发与CMake构建，实现了仿真数据的实时渲染与交互控制，同时兼容主流车载通信协议的模拟数据接入。项目兼顾了仿真精度与运行效率，可作为车载仪表算法验证、界面开发的基础平台，也为智能网联汽车相关仿真研究提供了轻量化的实验载体。
## 环境要求
- 操作系统: Ubuntu 22.04- 编译器: gcc 11.3.0
- CMake: 3.22.1
- 其他依赖: - MuJoCo 2.3.7（物理引擎核心库）
- Eigen 3.4.0（矩阵运算库）
- GLFW 3.3.8（图形窗口管理）
- OpenCV 4.8.0（图像渲染辅助）
- Boost 1.74.0（通用工具库）
## 编译
cd build
cmake --build . -j$(nproc) 
### 运行  
export LIBGL_ALWAYS_SOFTWARE=1
./bin/mjpc --task=Simple_car

##⽂件说明
-  dashboard_data.h ：数据获取模块，负责从MuJoCo仿真环境读取车辆状态参数
-  dashboard_render.h ：渲染模块，实现仪表盘图形界面的绘制与动画效果
-  mpc_controller.cpp ：MPC控制算法核心，计算车辆的最优控制指令
-  main.cpp ：程序入口，整合数据获取、控制算法与渲染模块的逻辑

## 已知问题
- 高帧率下仪表盘渲染存在轻微卡顿，需优化OpenGL绘制的批处理逻辑
- 新能源车型的电池电量仿真模型精度不足，待结合真实电池特性曲线修正
- 部分特殊工况（如急刹、极限转弯）下，MPC算法的响应延迟超过预期阈值
